##  О проекте

Данный проект представляет собой полный пайплайн предобработки телеметрических данных нефтяных скважин с использованием библиотеки **NumPy**. 

В работе реализованы:
- ✅ Векторизованная обработка больших данных (~2 млн записей)
- ✅ Групповая нормализация и очистка выбросов
- ✅ Анализ временных рядов (скользящие окна, лаги, производные)
- ✅ Feature Engineering с учётом предметной области
- ✅ Оптимизация памяти через строгую типизацию

---

## 🎓 Цель и задачи

### Цель
Освоить методы высокопроизводительной обработки данных в Python с применением векторизованных вычислений NumPy на примере реальных телеметрических данных нефтегазовой отрасли.

### Основные задачи
1. Загрузка и первичный аудит данных
2. Векторизованная фильтрация аномалий
3. Групповая Z-score нормализация
4. Сглаживание временных рядов (O(N) алгоритмы)
5. Инженерия признаков (Feature Engineering)
6. Условная агрегация по группам
7. Анализ временных сдвигов (лаги)
8. Робастная замена выбросов (IQR-метод)
9. Логическая верификация данных
10. Частотный анализ и сжатие категорий

Требования к данным
- Исходный файл: data.csv (не включён в репозиторий из-за размера)
- Формат: CSV с разделителем ,
- Кодировка: UTF-8
- Столбцы: ts, well_id, press, temp, vib, fluid

Запуск обработки
-Поместите data.csv в корень проекта
-Откройте numpy.ipynb в Jupyter Notebook
-Запустите все ячейки последовательно (Cell → Run All)
-Итоговый массив сохранится в data_final.npy

🔬 Этапы обработки данных
1️⃣ Загрузка и первичный аудит
Чтение CSV в структурированный массив (dtype)
Оптимизация памяти: int16, float32
Подсчёт NaN и Inf значений
Порог качества: < 3% пропусков
2️⃣ Векторизованная очистка аномалий
Булевы маски для физических ограничений
press < 0 → 0
fluid > 100 → 100
np.where для безопасной замены
3️⃣ Групповая нормализация (Z-score)
Расчёт статистики внутри каждой скважины
Формула: z = (x - μ) / σ
Итог: mean ≈ 0, std ≈ 1
4️⃣ Скользящее окно и перепады
Алгоритм O(N) через кумулятивные суммы
Окно k = 35 для сглаживания вибрации
np.diff для расчёта скорости изменения давления
Новый признак: press_abs_diff (сила перепада)
5️⃣ Feature Engineering
Созданы производные признаки:
vib_intensity = vib / press — нагрузка на оборудование
temp_press_ratio = temp / press — термобарический индекс пласта
Защита от деления на ноль: + 1e-8
Замена Inf/NaN → 0
6️⃣ Условная агрегация
Критический режим: press > 160 И vib > 60
Составная булева маска
Расчёт mean, median, P90 только для критических зон
7️⃣ Лаговые признаки
lag = 1 через np.roll
vib_diff = vib[t] - vib[t-1]
Анализ знаков: np.sign → рост/падение/стабильность
8️⃣ IQR-очистка выбросов
Межквартильный размах внутри групп
Границы: [Q1 - 1.5·IQR, Q3 + 1.5·IQR]
Замена выбросов на медиану группы
Робастность к экстремальным значениям
9️⃣ Логическая верификация
Проверка физических ограничений
Составная маска нарушений
Финальная очистка перед анализом
🔟 Сжатие категорий
Порог редкости: 1% от общего объёма
Объединение редких скважин в OTHER (код 255)
Векторизованная замена: np.isin + np.where
